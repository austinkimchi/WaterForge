Inserting New Tasks
31
1. Insert a new blank row where you want the new task to be
32
2. Copy the entire row you want to use from the set of template rows
33
3. Paste the row you copied on top of the blank row you just inserted
34
4. Copy and paste the WBS cell separately, based on the level (1, 2.1, 3.2.1, 4.3.2.1)
35
36
 - When inserting new rows, you must copy and paste an entire row, because the cells of the Gantt chart area are formulas.

Changing the WBS Level in the WBS Column
39
 - The WBS numbering uses a different formula for each level, but the formula does not reference any other cell in the row. So, you can copy and paste just the WBS cell that you want to use.

 - If you leave a blank cell above a WBS number, the numbering will reset to 1.x.x. The formulas are meant for convenience, but you can manually enter them if you need to.

Category Tasks
46
 - You can use tasks that are just labels, but it can be even more useful for a category task to display the minimum Start date and maximum End date of its sub tasks. This can be done using =MIN(range_of_startdates) and =MAX(range_of_enddates). An example template row is provided, but you will need to update the MIN() and MAX() formulas.

Creating Task Dependencies
49
50
 - You can enter the Start date manually, or define task dependecies using a formula. Below are the most common options for defining the Start date:
51
52
A.Enter the date manually (e.g. 1/3/2015)
53
B.Reference the Project Start Date (e.g. =$E$4 )
54
C.Set the Start date to the next Work Day after another task's End date.
55
 - Use the formula =WORKDAY(enddate,1) where enddate is the reference to the End date of a predecessor task.
56
 - For multiple predecessors, the formula would be =MAX(WORKDAY(enddate1,1),WORKDAY(enddate2,1))
57
58
D.Set the Start date to the next Calendar Day after another task's End date.
59
 - This formula is very simple: =enddate+1
60
 - For multiple predecessors, the formula would be =MAX(enddate1,enddate2,enddate3 )+1
61
62
E.Set the Start date to a number of days before or after another date.
63
 - This formula is just like the one in C or D, except that in place of the "1" you enter the number of days, such as =WORKDAY(enddate,5) or =WORKDAY(startdate,-5)

Q:What is the best way to Print?
67
A:First, select all of the rows you want to print. Then, in the Print Settings, choose "Selection" and check "No Gridlines". Fit to width and print in landscape.
68
You also may want to hide the Days Done, Days Left, and Color columns prior to printing.
69
70
Q:How do I only show Monday-Friday in the chart area?
71
A:You can hide the columns that show the weekends.
72
73
Q:How do I print the entire range of dates for my project?
74
A:You would first need to add more columns to the displayed chart area. You can insert more columns to the right of the chart area and then copy and paste columns (7 at a time) to extend the display.
75
76
Note: The more columns you add to the right of the Gantt chart, the slower the recalculation speed will be, because of the number of additional formulas.

Q:How do I calculate the %Complete for a Summary task?
79
A:The %Complete for a summary task can be calculated from its sub tasks using the formula below, where "workdays" is a reference to the range of work days and "complete" is a reference to the %complete for each of the subtasks.
80
 =ARRAYFORMULA( SUMPRODUCT( workdays, complete ) / SUM ( workdays ) )

Q:The Start date, End date, or %Complete for a Level 1 task is wrong. How do I fix it?
83
A:When using =MIN(), =MAX(), and =SUMPRODUCT(), it is easy for the references to get messed up if you move rows around or insert new rows. You should verify and fix these formulas if they are not referencing the correct ranges.
